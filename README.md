# CS 499 Computer Science ePortfolio

Welcome to my CS 499 Computer Science capstone ePortfolio. This portfolio represents my growth throughout the Computer Science program and includes my professional self-assessment, code review, enhanced artifacts, and narratives for each enhancement category.

## Professional Self-Assessment

Completing the Computer Science program and building this ePortfolio has helped me see how much I have grown as a developer. When I first created some of these projects, my main goal was usually to make sure the program worked and met the assignment requirements. As I moved through the program, I started looking at software differently. I began thinking more about structure, readability, validation, testing, databases, security, and how another developer or user might understand the project later. This ePortfolio represents that growth because it shows not only the original work I completed, but also how I was able to return to older projects and improve them with a stronger and more thoughtful approach.

One of the biggest skills I developed throughout the program was learning how to communicate technical work more clearly. In computer science, writing code is only one part of the process. A developer also needs to explain what the code does, why certain choices were made, and how the work solves a problem. Through my code review video, enhancement narratives, and project documentation, I practiced explaining my decisions in a way that could make sense to both technical and nontechnical audiences. This helped me understand that communication is part of professional software development, not something separate from it.

The code review process also helped me think more like a developer working in a team environment. Even though I completed much of the capstone work independently, the code review gave me the chance to look at my older projects from the perspective of a teammate, manager, or future developer. I had to explain the existing functionality, identify weaknesses, and describe the enhancements I planned to make. This helped me practice giving useful technical feedback and thinking about how code organization, comments, and design decisions can affect other people who may need to read, maintain, or build on the project later.

My software engineering skills are shown through my CS 230 Game App enhancement. This project manages games, teams, and players using object-oriented design. In the enhanced version, I improved the structure of the program by strengthening validation, protecting internal lists, cleaning up the shared Entity class, and making the singleton GameService easier to follow. This artifact shows my growth because I was able to take a project that already worked and make it cleaner, safer, and easier to maintain. I also learned that good software design is not just about making a program run. It is also about making the code organized, dependable, and easier for someone else to understand.

My algorithms and data structures skills are shown through my CS 300 Advising Program enhancement. This project loads course information, stores course data, sorts courses, and allows a user to search for specific course details. In the enhanced version, I improved file validation, prerequisite checking, menu input handling, and course searching. I used a map to organize courses by course number, a vector to store prerequisites, and sorting to display the courses in a clearer order. This enhancement helped me think more carefully about how data should be stored, searched, validated, and presented to the user.

The CS 300 project also helped me better understand trade-offs in software design. I wanted to improve the program without completely changing what it was supposed to be. Because of that, I focused on changes that made the program more reliable and easier to use while still keeping the original purpose of the advising program. I also learned that testing is not just about seeing whether the code compiles. It is also about checking how the program behaves when a file is missing, when a user enters the wrong kind of input, or when course data contains an issue.

My database skills are shown through my CS 340 Animal Shelter Dashboard enhancement. This project connects a dashboard to MongoDB and displays animal shelter records in a table, chart, and map. For the enhancement, I improved the database connection setup, added support for environment variables, strengthened the CRUD module, improved filtering, and made the dashboard safer when data was missing or unavailable. This project helped me understand that database development is not only about connecting to a database and displaying records. A stronger database application also needs to handle connection problems, empty results, invalid input, and clear user feedback.

Security became a bigger part of how I think about development during this program. Earlier on, I mostly thought about security as its own separate topic. Now I understand that a security mindset should be part of regular software design. In my enhancements, I used validation, protected internal data, improved error handling, and avoided relying only on hardcoded database credentials. These changes helped me think more about what could go wrong and how the software should respond. Even when a project is not specifically a security project, careful validation, safer data handling, and clearer error handling still help make the application more dependable.

The artifacts in this ePortfolio fit together because each one shows a different part of my growth in computer science. The CS 230 Game App shows my growth in software design, object-oriented programming, validation, and maintainability. The CS 300 Advising Program shows my growth in algorithms, data structures, sorting, searching, input handling, and testing. The CS 340 Animal Shelter Dashboard shows my growth in database development, MongoDB, CRUD operations, filtering, and data-driven applications. Together, these artifacts show that I can work across multiple areas of computer science and improve existing projects in meaningful ways.

Overall, this ePortfolio shows the kind of developer I am becoming. I am someone who can learn, troubleshoot, reflect, and keep improving. I have become more comfortable reading older code, finding weak spots, and making changes that actually improve the quality of a project. I have also learned that professional software development is about more than writing code that runs. It is about building software that is organized, secure, useful, maintainable, and understandable. As I move forward in the computer science field, I want to continue building on these skills and focus on creating software that solves real problems while being thoughtful about design, users, data, and security.
